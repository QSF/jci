Está com problema na hora de editar um usuário, devido ao seu relacionamento com campo e publico.
Como a gente tem que pesquisar o publico(campo) para carregar pegar os dados completos, o public se torna managed(se não souber disso, vai na doc do doctrine).
O fato de torná-lo gerenciável cria um bug na hora de usar o update, pois o entityManager pelo visto(a gente não sabe ao certo o que rola) encontra que este usuário é novo e assim lança um erro.
Para resolver isso, colocamos um clear antes de usar o update, o que não é adequado, pois o clear é algo exclusivo do Doctrine, ou seja, nosso controller nem deveria saber da existência do doctrine.
Caso a gente ganhe, ou os bugs posteriores nos vençam, iremos estudar isso a fundo; caso contrário, ficará como está.
